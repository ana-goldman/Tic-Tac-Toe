## Overview

This project is a Node.js workspace that will create a simple tic-tac-toe game in HTML, CSS, and JavaScript. The server code works and should not be tampered with. Each time you are prompted, do _only_ exactly the tasks requested of you by the user, and be conservative in the changes you make.

## Preferences

- The user has intermediate experience with JavaScript and Node.js.
- Keep explanations concise and focused on the requested change.
- Use small helper abstractions when they clearly reduce repetition.
- Prefer conservative, scoped edits over broad refactors.

## Workflows

When I say "next":

1. Read spec.md and find the next unchecked item (- [ ]).
2. Implement ONLY what is needed for this item in game.js and/or index.html.
3. Add all changes and commit to the Git repository, leaving a descriptive, verbose commit message.
4. Check off the item in spec.md (change - [ ] to - [x]).
5. Ask me ONE short question about what just changed (to verify that I understand).
6. Add a brief section to README.md explaining this feature (2 - 3 sentences max).
7. STOP and wait for my next instruction.

IMPORTANT: Only do ONE checklist item per "next" command. Do not continue to the next item automatically. Instead, stop and report what you've accomplished, checking back in with the user for feedback.
